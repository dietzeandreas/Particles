Damit ihr im Programm etwas verändern könnt, muss zunächst Visual-Studio und danach XNA 4.0 installiert werden.
Über VS kann das Programm dann auch manuel gestartet werden. Falls VS und XNA bereits installiert sind, kann die Anwendung
ebenfalls über die Datei ...\2DParticelEngine\bin\x86\Debug\2DParticelEngine.exe gestartet werden.

Wichtig:
Auf den Hochschulrechnern ist XNA 4.0 sowie Visual Studio 2010/2012/2013 bereits installiert. 
Hier braucht ihr demnach nichts machen, außer dass Programm über einen der oben erwähnten Wege zu starten.

Die folgende Installationsbeschreibung ist für Windows-Systeme, auf denen weder XNA oder Visual Studio installiert ist:

SETUP von SetupParticles
Falls ihr kein Interesse habt, im Quellcode selbst herumzuforschen, kann das Programm auch ohne 
die oben aufgelisteten Schritte gestartet werden.


1) Setup ausführen 
Während der Installation wird geprüft, ob sich das XNA-Framework auf dem Rechner befindet.
Ist dies nicht der Fall, wird es automatisch heruntergeladen. Außer einer kleinen Verknüfpung im Pfad
(C:\Users\User\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\2DParticelEngine)
wird nichts weiter Installiert. Auch nach der Installation, befinden sich alle benötigten Dateien im 
Extraktionsverzeichnis.




Deinstallation
Über den Software-Manager von Windows deinstallieren (wie bei jedem anderem Programm auch).
